# Разворот Cloudflare Worker для обхода блокировки Supabase в РФ

Эта инструкция нужна, если на iPhone в России без VPN не загружаются данные
ячеек (`TypeError: Load failed`). После выполнения шагов приложение будет
работать без VPN.

**Время на разворот:** ~15 минут. **Стоимость:** бесплатно (лимит CF — 100 000 запросов/день).

---

## Шаг 1. Регистрация на Cloudflare (~3 мин)

1. Открыть [https://dash.cloudflare.com/sign-up](https://dash.cloudflare.com/sign-up).
2. Указать email + пароль → подтвердить email.
3. **Ничего больше делать не нужно** — домен подключать не требуется,
   Worker получит адрес на `*.workers.dev` сразу из коробки.

---

## Шаг 2. Создание Worker (~5 мин)

1. В левом меню: **Workers & Pages**.
2. Нажать **Create** → выбрать **Workers** → **Hello World**.
3. Имя сервиса: `dvintsev-proxy` (или любое другое латиницей).
4. Нажать **Deploy**. CF создаст «hello world» Worker за пару секунд.
5. После деплоя нажать **Edit code** (или «Visit page» → потом «Edit Worker»).
6. **Откроется редактор кода:**
   - Удалить весь дефолтный код (Cmd+A → Backspace).
   - Открыть локально файл `cloudflare-worker.js` (в корне проекта парковки).
   - Скопировать весь его содержимое и вставить в редактор CF.
7. Нажать **Save and deploy** (правый верхний угол).

---

## Шаг 3. Получить URL Worker (~1 мин)

1. Закрыть редактор → вернуться к карточке Worker.
2. На странице сервиса вверху написано:
   ```
   Routes
   dvintsev-proxy.<your-account-name>.workers.dev
   ```
3. Скопировать этот адрес. Это **ваш персональный URL прокси**.
   Пример: `dvintsev-proxy.aleksandrzidilev.workers.dev`.

**Проверка:** откройте этот URL в Safari — должна появиться страница
вида «No API key found in request» (так и должно быть — значит Worker жив
и проксирует на Supabase).

---

## Шаг 4. Подключить прокси в parking.html (~2 мин)

1. Открыть `parking-public/parking.html` в любом редакторе.
2. Найти строку (около №2575):
   ```js
   const SB_PROXY_URL = '';
   ```
3. Вписать туда полный адрес worker с `https://`:
   ```js
   const SB_PROXY_URL = 'https://dvintsev-proxy.aleksandrzidilev.workers.dev';
   ```
   ⚠️ **Без слэша на конце.**

---

## Шаг 5. Деплой на GitHub Pages (~1 мин)

В терминале:

```bash
cd parking-public
git add parking.html
git commit -m "Enable Cloudflare Worker proxy for Supabase"
git push origin main
```

GitHub Pages обновит сайт за ~60 секунд.

---

## Шаг 6. Проверка на iPhone

1. Закрыть полностью вкладку Safari с приложением парковки.
2. **Выключить VPN** (если был включен).
3. Открыть свежую вкладку → ввести `https://apzhidilev-wq.github.io/dvintsev-parking/parking.html`.
4. Залогиниться.
5. Данные должны подтянуться **без VPN**.

Двойной тап по «Облако ●» покажет диагностику — в первой строке
должно быть `URL: proxy` (вместо `direct`).

## Если что-то пошло не так

### iPhone всё ещё не подтягивает данные

1. Двойной тап по облаку → проверить, что в первой строке `URL: proxy`.
   - Если `direct` — значит `SB_PROXY_URL` пустой, GitHub Pages не успел
     обновиться, либо в Safari закеширован старый JS. Подождите минуту,
     полностью закройте вкладку.
2. Открыть в Safari ваш Worker URL напрямую — он должен отвечать.
3. На вкладке Cloudflare → Workers → ваш сервис → вкладка **Logs**
   (Real-time) — там видно каждый запрос. Если приходят запросы
   и возвращают 200 — прокси работает.

### Ошибка "Forbidden origin" в логах CF

В `cloudflare-worker.js` есть массив `ALLOWED_ORIGINS`. Если ваш сайт
размещён по другому адресу (не `apzhidilev-wq.github.io`), добавьте
свой адрес в этот массив и **передеплойте Worker**:

```js
const ALLOWED_ORIGINS = [
  'https://apzhidilev-wq.github.io',
  'https://your-other-site.com',  // ← добавили
  'http://localhost:8000'
];
```

### Realtime не работает (статус «Облако ×»)

Worker по идее проксирует и WebSocket. Если не работает — это самая
сложная часть. Можно временно оставить как есть: `sbPullAllCells()`
по тапу будет работать через прокси, а Realtime будет падать тихо.

---

## Откат на прямой Supabase

Если нужно вернуться к прямому подключению (например, везде VPN или
переехали из РФ):

```js
const SB_PROXY_URL = '';  // ← очистить
```

→ commit → push. Worker можно не удалять — он не мешает.

---

## Безопасность

- Worker **не хранит никаких секретов**. Он просто переписывает Host-заголовок.
- Anon-ключ Supabase передаётся клиентом так же, как и при прямом обращении —
  это публичный ключ, его нормально показывать в HTML.
- Все RLS-правила Supabase продолжают работать — Worker не имеет привилегий.
- Если кто-то узнает ваш Worker URL — он сможет дёргать Supabase через него,
  но не более того, чем мог бы напрямую (RLS защищает данные).

Если хотите ограничить доступ только своим сайтом — `ALLOWED_ORIGINS` уже
настроен на это.
