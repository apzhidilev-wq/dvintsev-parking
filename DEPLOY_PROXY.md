# Подробная инструкция: Cloudflare Worker для обхода блокировки Supabase в РФ

После выполнения шагов приложение будет работать на iPhone в России **без VPN**.

| Параметр | Значение |
|---|---|
| Время на разворот | ~15 минут |
| Стоимость | бесплатно |
| Что нужно | email + один раз ввести имя пользователя |
| Что НЕ нужно | свой домен, банковская карта, телефон |
| Лимиты бесплатного тарифа | 100 000 запросов/день, ~10 ГБ трафика — для вашей нагрузки бесконечно |

---

## Шаг 1. Регистрация на Cloudflare

1. Открыть [https://dash.cloudflare.com/sign-up](https://dash.cloudflare.com/sign-up).

2. На экране регистрации:
   - **Email** — любая ваша почта.
   - **Password** — придумать пароль (минимум 8 символов, буквы + цифры).
   - Поставить галочку «Я согласен с условиями».
   - Нажать **Sign Up**.

3. На почту придёт письмо от Cloudflare с темой «Verify your email».
   Открыть его, нажать **Verify email**.
   *(Если письма нет — проверить «Спам». Если кнопка не работает —
   скопировать ссылку из письма и открыть в Safari.)*

4. После подтверждения email откроется главная панель Cloudflare
   (`dash.cloudflare.com`). Появится экран **«Add a website or application»**
   или приветственный квиз — **закрыть его / нажать "Skip"**, домен нам
   не нужен.

5. (Опционально) Cloudflare может попросить включить 2FA. Можете включить
   позже — для разворота Worker это не критично.

---

## Шаг 2. Открыть раздел Workers

В Cloudflare регулярно меняется интерфейс. Возможны два варианта:

### Вариант A (новый интерфейс, 2025+)

1. В левом меню панели — пункт **Compute (Workers)** или просто **Workers**
   (значок жёлтого квадрата с молнией).
2. Нажать **Workers & Pages** или сразу **Workers**.

### Вариант B (если интерфейс старый)

1. В левом меню — **Workers & Pages**.

### Если вообще не находите

Откройте напрямую: [https://dash.cloudflare.com/?to=/:account/workers-and-pages](https://dash.cloudflare.com/?to=/:account/workers-and-pages)

---

## Шаг 3. Подтвердить subdomain `workers.dev` (только при первом Worker)

При первом заходе в Workers Cloudflare попросит выбрать **subdomain**
(имя пользователя), например `aleksandrzidilev` — все ваши Worker'ы будут
жить по адресу `<имя-worker>.aleksandrzidilev.workers.dev`.

1. Появится поле «Choose your workers.dev subdomain».
2. Ввести любое имя латиницей (3-20 символов, буквы/цифры/дефис).
   Это имя нельзя поменять потом — выбирайте осознанно.
3. Нажать **Set up**.
4. Cloudflare проверит уникальность и подтвердит.

---

## Шаг 4. Создать Worker

1. На странице Workers & Pages нажать большую синюю кнопку
   **Create application** или **Create**.

2. Появится экран с тремя плитками:
   - **Workers** ← выбрать эту
   - Pages
   - Pages from Git

   Если плиток нет — сразу выберите **Hello World** template.

3. На следующем экране **Hello World** Worker:
   - Вверху поле **Name** — ввести `dvintsev-proxy` (или любое другое
     латиницей, без пробелов).
   - Внизу **Deploy** (или **Save and Deploy**).

4. Cloudflare деплоит шаблон-«Hello World» за 5-10 секунд. Появится экран:
   ```
   Successfully deployed
   Visit ↗ dvintsev-proxy.aleksandrzidilev.workers.dev
   ```

5. Нажать кнопку **Edit code** или **Continue to Project** → **Edit code**.
   Откроется онлайн-редактор кода.

---

## Шаг 5. Заменить код Worker

1. В редакторе слева вы видите файл `worker.js` с дефолтным кодом
   (примерно `export default { fetch() { return new Response("Hello World!") } }`).

2. **Выделить весь код** (поставить курсор в редактор → нажать
   **Cmd + A** на Mac или **Ctrl + A** на Windows).

3. **Удалить** (нажать Backspace или Delete).

4. Открыть локально файл `cloudflare-worker.js` (он лежит в корне проекта
   парковки рядом с `parking.html`). Простой способ:
   ```bash
   cat /путь/к/parking-public/cloudflare-worker.js | pbcopy
   ```
   *(на Mac это копирует содержимое в буфер обмена)*.

   Или просто откройте файл в любом редакторе → выделить всё → копировать.

5. **Вставить** в редактор Cloudflare (Cmd + V / Ctrl + V).

6. Справа вверху — кнопка **Save and deploy**. Нажать.

7. Cloudflare задеплоит за пару секунд. Появится зелёный значок
   «Deployed» с временем.

---

## Шаг 6. Проверить, что Worker жив

1. Скопируйте URL Worker (виден на странице, формат
   `https://dvintsev-proxy.<ваш-account>.workers.dev`).

2. Откройте этот URL в новой вкладке Safari/Chrome.

3. **Что должно появиться:**
   - Текст вида:
     ```json
     {"message":"No API key found in request","hint":"No `apikey` request header or url param was found."}
     ```
   - Это **правильно**: Worker проксирует на Supabase, Supabase отвечает,
     что без API-ключа доступа нет. Значит цепочка работает.

4. **Если появилось другое:**
   - «1101 Worker threw exception» / «1102 Worker exceeded CPU time» —
     ошибка в коде. Вернуться в редактор → проверить, что вставили
     именно содержимое `cloudflare-worker.js` целиком.
   - «525 SSL handshake failed» — подождать минуту, обновить.
   - «Network error» / «Не удалось открыть» — проверить, что URL
     правильный (включая `https://`).

## Шаг 7. Подключить прокси в `parking.html`

1. На вашем Mac, где лежит репозиторий проекта, открыть файл
   `parking-public/parking.html` в любом редакторе (VS Code, TextEdit и т.д.).

2. Найти строку (примерно №2575, можно через **Cmd+F**: `SB_PROXY_URL`):
   ```js
   const SB_PROXY_URL = '';
   ```

3. Заменить на ваш URL Worker:
   ```js
   const SB_PROXY_URL = 'https://dvintsev-proxy.aleksandrzidilev.workers.dev';
   ```
   ⚠️ **Без слэша `/` на конце.**

4. Сохранить файл.

## Шаг 9. Проверка на iPhone

1. **Выключить VPN** на iPhone, если он включён.

2. Открыть Safari → проверить, что адрес `apzhidilev-wq.github.io/dvintsev-parking/parking.html`
   ещё открыт. Если открыт — **закрыть вкладку полностью** (свайп вверх в
   режиме вкладок), и открыть свежую.

3. В свежей вкладке ввести адрес заново:
   `https://apzhidilev-wq.github.io/dvintsev-parking/parking.html`

4. Залогиниться.

5. Через 5–10 секунд должны появиться все ячейки и статистика.

6. **Двойной тап по «Облако ●»** в правом верхнем углу — должна появиться
   диагностика. **Первая строка должна быть:** `URL: proxy`.
   Если там `direct` — значит GitHub Pages ещё не обновил JS, либо Safari
   закешировал. Подождите минуту, закройте вкладку полностью, откройте
   свежую.

## Шаг 10. Мониторинг (опционально)

В Cloudflare на странице вашего Worker есть две вкладки:

### Logs (Real-time)
1. Открыть вкладку **Logs** → **Begin log stream**.
2. На iPhone обновить страницу — в логах CF появятся каждый запрос
   (URL, статус, время ответа).
3. Так видно, что работает, и насколько быстро.

### Metrics
- Графики запросов в день, ошибок, длительности.
- Если приближаетесь к лимиту 100 000/день — нужно тарифицировать.
  Для нашей задачи это нереально превысить.

## Возможные проблемы и решения

### «На iPhone всё равно красное облако с крестиком»

Realtime WebSocket через Worker — самая хрупкая часть. Если только
real-time не работает, но обычный pull (по тапу) работает — это
приемлемо. Данные подтянутся, просто синхронизация между устройствами
будет не мгновенная, а по обновлению.

### «Forbidden origin» в логах CF

В `cloudflare-worker.js` есть массив `ALLOWED_ORIGINS`. Если у вас другой
адрес сайта (например, развернули копию на другом домене), добавьте его:

```js
const ALLOWED_ORIGINS = [
  'https://apzhidilev-wq.github.io',
  'https://example.com',  // ← добавили
  'http://localhost:8000'
];
```

После изменения — **снова Save and Deploy** в Cloudflare.

### «1101 Worker threw exception» при открытии URL Worker напрямую

Откройте редактор Worker → перепроверьте, что код вставлен **целиком**
(особенно — что не пропали закрывающие скобки в конце файла). Сверьтесь
с локальным `cloudflare-worker.js`.

### Хочется проверить с обычного Mac, что прокси работает

В консоли DevTools (F12) Mac:
```js
fetch('https://dvintsev-proxy.aleksandrzidilev.workers.dev/rest/v1/parking_cells?select=*&limit=5', {
  headers: {
    'apikey': 'eyJhbGc...' // SB_ANON_KEY из parking.html
  }
}).then(r => r.json()).then(console.log)
```

Должен вернуться массив из 5 ячеек.

### Хочется откатиться к прямому Supabase

В `parking.html`:
```js
const SB_PROXY_URL = '';  // ← очистить
```
→ commit → push.

Worker в Cloudflare можно не удалять — он не мешает и не тратит ресурсы,
если к нему никто не обращается.

## Безопасность

- Worker **не хранит никаких секретов**. Его код виден всем, кто знает URL.
- **Anon-ключ Supabase**, который передаётся через Worker — это публичный
  ключ, его нормально показывать в HTML. Защита данных — на уровне Supabase
  RLS-политик (которые уже настроены).
- `ALLOWED_ORIGINS` ограничивает, с каких сайтов разрешено дёргать прокси.
  Если кто-то узнает ваш Worker URL и попробует использовать его со своего
  сайта — браузер заблокирует через CORS. Прямые запросы вне браузера
  (curl, скрипты) могут пройти, но они и так могут идти напрямую к Supabase.
- Если хотите ещё сильнее ограничить — можно потребовать секретный
  заголовок (см. комментарии в `cloudflare-worker.js`, но это не нужно для
  обычного использования).

---

## Что делать, если CF Worker лимит превышен

100 000 запросов/день — это ~70 запросов/минуту. Чтобы превысить,
нужно несколько активно работающих пользователей, обновляющих страницу
каждые секунды.

Если это случится:
1. Cloudflare пришлёт письмо предупреждения.
2. Tariff **Workers Paid**: $5/мес → 10 миллионов запросов/мес.
   Это всё равно дёшево.
3. Альтернатива: создать второй Worker на втором аккаунте и
   распределять нагрузку.

Реалистично — для одного БЦ парковки лимит **никогда** не превысится.
